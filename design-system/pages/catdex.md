# Catdex Page Design Rules

Catdex is the collection view. It should feel like a found-cat archive, not a file manager.

## Structure

- Use dynamic catdex numbers as collection identity.
- Grid cards can be denser than Home, but still use postcard/sticker materials.
- Each entry should prioritize the cat image, then date/location metadata.
- Filters should feel like archive tools, not dashboard controls.

## Card Treatment

- Use off-white paper surfaces.
- Use ink borders and small blue accents.
- Prefer cat image and catdex number over long text.
- Location can appear as a small corner clue.
- Cat names use `FoundCat Round`; archive numbers use `FoundCat Number`.

## Empty State

Invite capture:

- `還沒有貓咪圖鑑`
- `拍下第一隻遇見的貓，牠會出現在這裡。`

## Avoid

- Spreadsheet-like lists.
- Overly rounded generic cards.
- Excessive filters before the user has enough cats.
