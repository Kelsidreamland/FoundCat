# Home Page Design Rules

Home is the emotional entry point. It should make the user want to collect the cat they just met.

## Required Structure

1. A1-style postcard header:
   - `轉角遇到貓`
   - warm tagline
   - postal mark as brand element

2. E1 primary visual:
   - latest cat base photo with warm app filter;
   - same cat cut out, slightly enlarged, sticker border, blue offset shadow;
   - catdex number label such as `CATDEX 028`;
   - number is dynamic, never hard-coded.

3. Today sticker strip:
   - shows additional cats from today;
   - cutout stickers use irregular white clipping borders;
   - empty state should invite capture, not explain features.

4. Quiet corner clue:
   - location/map preview appears as secondary context;
   - do not make the map the main content on Home.

5. Bottom navigation:
   - icon-first;
   - capture remains central primary action.

## Typography

- Cat names use `font-cat-display` / `FoundCat Round`.
- `W-` and `No.` numbers use `font-cat-number` / `FoundCat Number`.
- Location and action copy remain in the body font so the card stays easy to scan.

## Empty State

When no cats exist:

- Keep the poster/card structure visible.
- Use quiet copy:
  - `今天還沒有遇見貓`
  - `拍下第一隻遇見的貓，牠會出現在這裡。`
- Do not show a blank dashboard.

## Avoid

- Large map on first viewport.
- Generic hero marketing copy.
- Daily Scrapbook labels.
- Text-heavy onboarding instructions.
