# FOUND CAT 企劃發想：備份、分享地圖與貓咪世界地圖

更新日期：2026-06-17  
狀態：腦力風暴中，作為 Notion / G-Brand 同步用母稿

## 一句話方向

FOUND CAT 先讓使用者像滑交友軟體一樣瀏覽全世界的公開貓卡，遇到喜歡的貓就收藏，也能拍下自己遇到的貓；長期成為一張由全世界貓友一起編輯、追蹤、收藏與照顧的貓咪世界地圖。

## 目前產品核心

- 使用者快速拍下遇到的貓。
- 將貓卡綁定到出沒地點。
- 在地圖上查看自己遇過的貓。
- 分享單張貓卡或某個地點的貓。
- 透過 Google Maps 導航到附近，實際找貓仍依賴備註、時間與出沒描述。

## 近期產品入口：全世界貓卡滑卡

首頁不再以「我的貓卡」作為預設主畫面，而是直接展示全世界公開貓卡。

核心互動：

- 首頁是貓卡左滑 / 右滑。
- 右滑：收藏這隻貓，並切換到下一張。
- 左滑：不收藏，直接切換到下一張。
- 左滑不代表不喜歡；產品語氣不設計「不喜歡貓」的選項。
- 卡片本身保持乾淨，不常駐顯示「右滑收藏 / 左滑下一隻」文字。
- 手勢說明只在第一次使用的新手教學短暫出現。

底部主導航：

- 左：我的貓卡。
- 中：拍貓 / 新增貓咪卡片。
- 右：地圖。

「我的貓卡」頁可以依使用者生活方式整理：

- 一般本地使用者：以國家內的縣市、城市、區域分組。
- 旅行者或長住海外使用者：以國家、城市、地區分組。
- 國旗或國家縮寫可作為輔助，但不要讓頁面變成花俏 icon 牆。

地圖頁保留完整地圖，並在同一頁切換：

- 我的地圖：我自己拍過或收藏的貓點。
- 大家的地圖 / 全世界地圖：所有公開貓點。

## 近期定位：帳號是備份，不是社群壓力

註冊功能一開始應該讓人感覺是「備份我的貓咪資料」，而不是立刻加入社群。

適合的入口語氣：

- 備份我的貓咪地圖
- 換手機也不會弄丟今天遇到的貓
- 把你的貓咪足跡保存起來
- 登入後可以同步貓卡、照片、地點與備註

這一階段的價值是留存與安心，不急著做追蹤、留言、好友或公開社群。

登入 / 備份入口建議：

- 不登入也能瀏覽全世界公開貓卡與公開地圖。
- 收藏公開貓卡時，先保存到本機。
- 收藏後可以柔性提示：「已收藏到我的貓卡。用 Email 備份後，換手機也不會弄丟。」
- 右上角可有小型備份 / 帳號入口，但不能搶首頁貓卡主視覺。
- 我的貓卡頁上方可放備份狀態：本機保存、已登入、可恢復雲端資料。
- 公開自己的貓到全世界地圖時才需要登入，原因是需要之後能修改或撤回。

## 拍貓後流程：直接回地圖

使用者存入一張新貓卡後，不再出現額外成功頁，也不重複顯示已存在地圖卡片上的功能。

確認流程：

1. 使用者拍照 / 選照片。
2. 裁切、填寫貓咪暱稱、地點與備註。
3. 按「存入我的貓卡」。
4. App 直接跳到貓咪地圖。
5. 地圖自動選中剛新增的貓點。
6. 剛新增的貓咪卡片直接打開。
7. 卡片保留原有分享、編輯地點、Google Maps 開啟等功能。
8. 卡片下方浮出淡玻璃提示：「想讓大家也看到這隻貓？登入後可以公開到全世界地圖，也能之後修改或撤回。」

這個流程的原則是：

- 不增加成功頁。
- 不重複分享功能。
- 不在拍照前要求登入。
- 只在「公開到全世界地圖」附近提示登入。

## 分享我的貓咪地圖

目前已經完成或接近完成：

- 分享單張貓卡。
- 分享某個地點的貓。
- 分享整張自己的貓咪地圖。
- 匯出 Google My Maps 可用的 CSV。
- 地圖點擊貓咪泡泡後可以查看貓咪資訊。
- 同一地點多隻貓會合併成群組泡泡，卡片內可切換。

尚未完成：

- 使用者帳號與雲端備份。
- 公開 / 私密設定。
- 多人共同編輯的貓咪世界地圖。

建議方向：

- FOUND CAT 自己的地圖是主體。
- Google Maps 只負責導航或匯出，不作為核心資料庫。
- 目前先提供匯出 CSV，讓進階使用者手動匯入 Google My Maps；KML 可留到後續需要時再加。

## 長期北極星：貓咪世界地圖

貓咪世界地圖不是一開始就強制社群化，而是建立在「使用者主動公開」之上。

概念：

- 世界各地使用者可以把自己遇到的貓公開到同一張地圖上。
- 地圖上可以看到不同城市、街區、店家、公園的貓點。
- 點進某個地點，可以看到該地點出沒過的多隻貓。
- 使用者可以收藏別人的公開貓點到自己的地圖。
- 使用者可以回報「我也看到牠了」。
- 長期可以形成「集體雲養貓」的體驗。

產品情境：

- 我想看看香港那隻貓最近在幹嘛。
- 我想知道曼谷某間咖啡店附近常出現哪些貓。
- 我想收藏朋友在日本拍到的街角貓。
- 我想追蹤某隻常在便利店門口出沒的貓。

## 同一隻貓的集體更新

長期可以讓多個使用者共同更新同一隻貓的近況。

早期不應完全依賴 AI 自動判斷，建議先做「疑似同一隻貓」的人工確認流程：

- 系統根據照片、地點、花色、時間提出可能相同的貓。
- 使用者看到提示：「這是同一隻貓嗎？」
- 使用者確認後，才合併到同一隻貓的時間線。

同一隻貓的時間線可包含：

- 第一次被發現的日期與地點。
- 最近一次被看見的時間與地點。
- 不同使用者拍到的照片。
- 出沒習慣、個性、照護狀態。
- 是否已剪耳 / TNR、是否有人固定餵養、是否疑似有人飼養。

這個方向不建議立刻做成底部第三入口。近期可以先把它設計成公開貓卡詳情往下延伸的模組：

- 其他人也看過這隻貓。
- 疑似同一隻貓，等待使用者確認。
- 近況更新時間線。
- 留言或補充性格描述。

## 商業化可能性

短期不建議以廣告作為核心。

Google AdSense / 廣告注意事項：

- 不應要求使用者「幫忙點廣告」。
- 不應用文案暗示點擊廣告可以支持本站，這可能違反廣告平台的無效流量政策。
- 若使用廣告或導購，應明確標示為廣告、贊助或推薦，不把點擊包裝成義務或支持行為。

更符合品牌的方向：

- 免費：本地保存、基本分享、少量雲端備份。
- 付費：大量雲端備份、高清照片、私密地圖、進階分享海報。
- 捐贈：支持 App 維護或特定地區流浪貓照護。
- 聯盟導購：貓條、貓砂、外出用品、寵物照護用品，以清楚標示的推薦區塊呈現。
- 志工合作：中途之家、TNR 團體、餵養志工、貓咖、寵物品牌。
- 長期功能：「請附近貓友探望牠」、「支持這個地區的餵養志工」。

使用者提出的長期想像：

- 線上支付小額金額，例如 5 元。
- 附近志工幫忙買貓條或食物去探望這隻貓。
- 志工拍照回報，讓遠端使用者知道牠最近過得如何。

這個方向有情感價值，但需要晚一點做，因為涉及信任、安全、支付與動物照護規範。

## 安全與倫理風險

貓咪世界地圖需要先處理公開精準度與動物安全。

重要風險：

- 家貓地址被公開。
- 私人住宅位置被暴露。
- 固定餵養點被過度打擾。
- 熱門貓點造成過度餵食或騷擾。
- 志工代餵涉及信任、退款、回報真實性與動物健康。

建議公開設定：

- 私人收藏：只有自己看得到。
- 模糊地區：只顯示街區或附近區域。
- 公開精準地點：適合商店、公園、咖啡廳等公共場域。
- 允許收藏：別人可以收藏到自己的地圖。
- 允許回報：別人可以補充「我也看到牠了」。

## 建議產品路線

### V0.1

穩定目前個人貓咪地圖、拍貓、地點、貓卡、分享海報與手機版體驗。

發布前精修應加入整體 UI 動效整理，重點不是增加花俏動畫，而是讓產品感覺更細緻、穩定且有品牌一致性。範圍包含：

- 首頁貓卡滑動的手感、回彈、收藏回饋。
- 頁面切換與底部導航的轉場節奏。
- 新增貓卡流程中的按鈕狀態、載入狀態與成功回饋。
- 地圖上貓咪卡片的彈出、收合、同地點切換動畫。
- 空狀態與載入狀態的品牌化微動畫。
- PWA 更新提示、登入/備份提示等浮層的進出場。

原則：動效要短、清楚、可預期；不影響手機效能，不遮擋主要操作，不為了動畫增加流程步驟。

### V0.2

改造首頁為全世界公開貓卡滑卡。右滑收藏、左滑下一張；底部主導航調整為「我的貓卡 / 拍貓 / 地圖」。加入選擇性註冊與雲端備份，註冊入口以「備份」為主，不以社群為主。

### V0.3

拍貓後直接跳地圖並打開剛新增貓卡。加入公開到全世界地圖的登入提示、公開 / 私密設定，以及分享整張自己的貓咪地圖。

### V0.4

完善公開到貓咪世界地圖、收藏別人的貓點、回報「我也看到牠了」。

### V1.0

發展成貓咪世界地圖與集體雲養貓平台，逐步加入志工、捐贈、探望回報與照護合作。

## 下一個需要討論的問題

公開到世界地圖時，一張貓卡最少需要哪些欄位，才會既好玩又不危險？

候選欄位：

- 貓咪照片。
- 貓咪暱稱。
- 出沒地點名稱。
- 模糊或精準位置。
- 最近看見時間。
- 出沒備註。
- 個性標籤。
- 照護狀態。
- 是否允許他人收藏。
- 是否允許他人回報近況。
