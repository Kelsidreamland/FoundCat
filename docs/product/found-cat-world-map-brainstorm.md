# FOUND CAT 企劃發想：備份、分享地圖與貓咪世界地圖

更新日期：2026-05-24  
狀態：腦力風暴中，作為 Notion / G-Brand 同步用母稿

## 一句話方向

FOUND CAT 先讓使用者備份自己的遇貓紀錄，長期成為一張由全世界貓友一起編輯、追蹤、收藏與照顧的貓咪世界地圖。

## 目前產品核心

- 使用者快速拍下遇到的貓。
- 將貓卡綁定到出沒地點。
- 在地圖上查看自己遇過的貓。
- 分享單張貓卡或某個地點的貓。
- 透過 Google Maps 導航到附近，實際找貓仍依賴備註、時間與出沒描述。

## 近期定位：帳號是備份，不是社群壓力

註冊功能一開始應該讓人感覺是「備份我的貓咪資料」，而不是立刻加入社群。

適合的入口語氣：

- 備份我的貓咪地圖
- 換手機也不會弄丟今天遇到的貓
- 把你的貓咪足跡保存起來
- 登入後可以同步貓卡、照片、地點與備註

這一階段的價值是留存與安心，不急著做追蹤、留言、好友或公開社群。

## 分享我的貓咪地圖

目前已經完成或接近完成：

- 分享單張貓卡。
- 分享某個地點的貓。
- 分享整張自己的貓咪地圖。
- 匯出 Google My Maps 可用的 CSV。
- 地圖點擊貓咪泡泡後可以查看貓咪資訊。
- 同一地點多隻貓會合併成群組泡泡，卡片內可切換。

尚未完成：

- 使用者帳號與雲端備份。
- 公開 / 私密設定。
- 多人共同編輯的貓咪世界地圖。

建議方向：

- FOUND CAT 自己的地圖是主體。
- Google Maps 只負責導航或匯出，不作為核心資料庫。
- 目前先提供匯出 CSV，讓進階使用者手動匯入 Google My Maps；KML 可留到後續需要時再加。

## 長期北極星：貓咪世界地圖

貓咪世界地圖不是一開始就強制社群化，而是建立在「使用者主動公開」之上。

概念：

- 世界各地使用者可以把自己遇到的貓公開到同一張地圖上。
- 地圖上可以看到不同城市、街區、店家、公園的貓點。
- 點進某個地點，可以看到該地點出沒過的多隻貓。
- 使用者可以收藏別人的公開貓點到自己的地圖。
- 使用者可以回報「我也看到牠了」。
- 長期可以形成「集體雲養貓」的體驗。

產品情境：

- 我想看看香港那隻貓最近在幹嘛。
- 我想知道曼谷某間咖啡店附近常出現哪些貓。
- 我想收藏朋友在日本拍到的街角貓。
- 我想追蹤某隻常在便利店門口出沒的貓。

## 同一隻貓的集體更新

長期可以讓多個使用者共同更新同一隻貓的近況。

早期不應完全依賴 AI 自動判斷，建議先做「疑似同一隻貓」的人工確認流程：

- 系統根據照片、地點、花色、時間提出可能相同的貓。
- 使用者看到提示：「這是同一隻貓嗎？」
- 使用者確認後，才合併到同一隻貓的時間線。

同一隻貓的時間線可包含：

- 第一次被發現的日期與地點。
- 最近一次被看見的時間與地點。
- 不同使用者拍到的照片。
- 出沒習慣、個性、照護狀態。
- 是否已剪耳 / TNR、是否有人固定餵養、是否疑似有人飼養。

## 商業化可能性

短期不建議以廣告作為核心。

更符合品牌的方向：

- 免費：本地保存、基本分享、少量雲端備份。
- 付費：大量雲端備份、高清照片、私密地圖、進階分享海報。
- 捐贈：支持 App 維護或特定地區流浪貓照護。
- 志工合作：中途之家、TNR 團體、餵養志工、貓咖、寵物品牌。
- 長期功能：「請附近貓友探望牠」、「支持這個地區的餵養志工」。

使用者提出的長期想像：

- 線上支付小額金額，例如 5 元。
- 附近志工幫忙買貓條或食物去探望這隻貓。
- 志工拍照回報，讓遠端使用者知道牠最近過得如何。

這個方向有情感價值，但需要晚一點做，因為涉及信任、安全、支付與動物照護規範。

## 安全與倫理風險

貓咪世界地圖需要先處理公開精準度與動物安全。

重要風險：

- 家貓地址被公開。
- 私人住宅位置被暴露。
- 固定餵養點被過度打擾。
- 熱門貓點造成過度餵食或騷擾。
- 志工代餵涉及信任、退款、回報真實性與動物健康。

建議公開設定：

- 私人收藏：只有自己看得到。
- 模糊地區：只顯示街區或附近區域。
- 公開精準地點：適合商店、公園、咖啡廳等公共場域。
- 允許收藏：別人可以收藏到自己的地圖。
- 允許回報：別人可以補充「我也看到牠了」。

## 建議產品路線

### V0.1

穩定目前個人貓咪地圖、拍貓、地點、貓卡、分享海報與手機版體驗。

### V0.2

加入選擇性註冊與雲端備份。註冊入口以「備份」為主，不以社群為主。

### V0.3

加入分享整張自己的貓咪地圖，以及公開 / 私密設定。

### V0.4

加入公開到貓咪世界地圖、收藏別人的貓點、回報「我也看到牠了」。

### V1.0

發展成貓咪世界地圖與集體雲養貓平台，逐步加入志工、捐贈、探望回報與照護合作。

## 下一個需要討論的問題

公開到世界地圖時，一張貓卡最少需要哪些欄位，才會既好玩又不危險？

候選欄位：

- 貓咪照片。
- 貓咪暱稱。
- 出沒地點名稱。
- 模糊或精準位置。
- 最近看見時間。
- 出沒備註。
- 個性標籤。
- 照護狀態。
- 是否允許他人收藏。
- 是否允許他人回報近況。
